# 中文用途索引

这个索引把仓库里的主要文件按“中文用途”解释清楚。代码文件本身尽量保留原路径和原扩展名，避免 GitHub 链接、脚本命令和自动化工具失效；在本地整理目录中会另外提供中文用途命名的副本。

## 项目入口

| 原文件 | 中文用途 |
| --- | --- |
| `README.md` | 项目总览与本地运行入口 |
| `AGENTS.md` | Codex / Jules 代理协作规则 |
| `.gitignore` | 防止上传缓存、输出、依赖和临时文件的忽略规则 |
| `.editorconfig` | 文本编码、换行和空白格式规则 |
| `.gitattributes` | Git 文本与二进制文件识别规则 |
| `package.json` | npm 快捷命令入口 |

## 本地整理位置

本地中文整理副本建议放在：

```text
E:\下载\codex
```

建议目录：

| 中文目录 | 用途 |
| --- | --- |
| `GitHub项目源码_保持可运行` | 保留原文件名和原目录结构，适合继续 Git / Jules / Codex 协作 |
| `中文用途整理版` | 用中文用途重命名的副本，适合人工查看和归档 |
| `交付文档` | Word、PDF、报告等成品文档 |
